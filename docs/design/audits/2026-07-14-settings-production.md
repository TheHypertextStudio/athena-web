## Findings resolved during the review

1. Mobile section navigation initially consumed most of the first viewport. It now becomes a compact horizontal destination row and remains a grouped sticky rail on wide layouts.
2. Provider status and actions collided on narrow Connections cards. Rows now wrap deliberately and preserve readable labels and action targets.
3. Settings controls inherited a 36px mobile height. Both Settings shells now enforce the rubric's 40px touch-target floor; a live 390px measurement returned zero undersized controls inside the Profile surface.
4. Profile photo and workspace logo were technically editable only through raw URL fields. Both now use an ordinary image picker with preview, replacement, removal, type/size validation, managed blob persistence, and application-owned error copy.
5. Workspace copy still used organization framing after the user-owned Settings split. The workspace shell and membership invitation now consistently describe workspace administration.
6. Google Calendar still linked through a legacy workspace URL and clipped its Sync action on narrow screens. It now has a first-class global Connections route, a preserving legacy redirect, and a responsive action layout.
7. The capture harness previously accepted 404 pages and could preserve corrupted Chromium frames. It now rejects non-2xx or signed-out navigation, recreates browser state per route, warms the compositor, and checks candidate PNG pixels before writing evidence.

Verdict: **SHIP BAR** — every dimension is at least 3 and every hard gate passes.
