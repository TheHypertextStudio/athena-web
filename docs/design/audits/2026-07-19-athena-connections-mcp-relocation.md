## Findings resolved in the first pass

1. **Google Tasks account row wrapped into an unreadable vertical stack at narrow widths** — `apps/web/src/components/settings/gtasks-accounts-section.tsx` (`GtasksAccountRow`). The scope-label span (`"Two-way · all lists"`) had no truncation, and its `shrink-0` action-button sibling squeezed the label column toward zero width. Fixed with the same `flex-wrap ... sm:flex-nowrap` pattern already proven in `integration-provider-card.tsx`, plus `truncate` on the scope label.
2. **`McpConnectorsSection`'s own header row had the identical bug**, newly prominent once it moved onto the Athena page — `apps/web/src/components/settings/mcp-connectors-section.tsx`. "Tools for Athena" + its description got squeezed by the non-wrapping "Add connector" button at 390px. Fixed with the same `flex-wrap items-start gap-4 sm:flex-nowrap sm:justify-between` pattern; re-verified by screenshot.

## Findings resolved in the follow-up pass

3. **Misdiagnosed as an app-shell bug; actually Next.js's dev-mode indicator.** Disabled with `devIndicators: false` in `apps/web/next.config.ts` — dev-only chrome that never shipped to production, but it was cluttering local design-review captures and reading as a real defect. Re-verified absent from all 8 shots in this pass.
4. **Terse "No tools yet." empty state** — `apps/web/src/components/settings/mcp-connectors-section.tsx`. Replaced with the same dashed-border, icon + explanation pattern the Google Tasks empty state already used: "No tools connected yet. Add a connector so Athena can act through the services you use." (branches to an admin-facing variant when `canManage` is false).
5. **No visual boundary between identity linking and workspace connections on the Connections page** — `apps/web/src/components/settings/connected-accounts-tab.tsx`. Added a `"Linked accounts"` label above the Google identity card, matching the label convention every other subsection on the page already used, so the page now reads as one consistent list of labelled sections instead of two conventions.

Verdict: **SHIP BAR** — every dimension ≥3, every gate green, zero open findings.
