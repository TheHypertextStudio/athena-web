## Findings resolved during the review

1. **Google Tasks account row wrapped into an unreadable vertical stack at narrow widths** — `apps/web/src/components/settings/gtasks-accounts-section.tsx` (`GtasksAccountRow`). The scope-label span (`"Two-way · all lists"`) had no truncation, and its `shrink-0` action-button sibling squeezed the label column toward zero width. Fixed with the same `flex-wrap ... sm:flex-nowrap` pattern already proven in `integration-provider-card.tsx`, plus `truncate` on the scope label.
2. **`McpConnectorsSection`'s own header row had the identical bug**, newly prominent now that it's the first thing on the Athena page — `apps/web/src/components/settings/mcp-connectors-section.tsx`. "Tools for Athena" + its description got squeezed by the non-wrapping "Add connector" button at 390px. Fixed with the same `flex-wrap items-start gap-4 sm:flex-nowrap sm:justify-between` pattern; re-verified by screenshot (button now drops below the text instead of colliding with it).

## Findings noted, not blocking

3. The app shell's floating profile-avatar chip overlaps trailing page content at 390px on both routes (pre-existing app-shell behavior, not touched by this change).
4. "Connected tools / No tools yet." is a terser empty state than the rubric's bar for a first-run surface; consider a one-line "next action" hint once MCP connector setup has a documented first step.
5. Connections mixes a bordered-card identity row (Google) with un-bordered section-header rows (Google Tasks/Calendar/Communication) — legible but two conventions on one page; worth reconciling in a future pass, not this one.

Verdict: **SHIP BAR** — every dimension ≥3, every gate green. Findings 3–5 are polish, tracked here rather than blocking this change.
